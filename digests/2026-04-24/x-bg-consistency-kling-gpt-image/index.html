GPT Image 2.0 × Kling 3.0 Omni で「アングルが変わっても背景一貫性」を保つ4段ワークフロー

AIアニメ/ドラマで頭痛だった「アングルが変わると背景・位置が崩れる」問題を、 Midjourney → ChatGPTで「場所の設定シート」生成 → GPT Image 2.0 → Kling 3.0 Omni の 4段ワークフローでほぼ解決。投稿当日で 1.7k+ views。

TL;DR

Setting / Conditions

Step 1 / Base Midjourney 等でベース画像を作成
Step 2 / Spatial schema ChatGPT に「場所の設定シート」を依頼。見えていない部分も含め部屋構造を言語化
Step 3 / Multi-angle 設定シート + 人物位置画像を ChatGPT (GPT Image 2.0) に渡し、アングル別画像を複数生成
Step 4 / Animate Kling 3.0 Omni でカスタム複数カット動画。アングル画像 + キャラシート参照 + カット毎のセリフ/動き指示
解像度の所感 GPT Image 2.0 が FullHD 以下でも Kling が補完。Seedance 2.0 より解像度影響が少ない
Author @kan_mi_no9（動画生成1年目、AIドラマ/アニメ制作）

Takeaways

  • 「見えていない部分も含めた空間の言語化」が一貫性キープの鍵。ChatGPT が起こす部屋構造シートが各画像生成の共通設計図になる
  • Kling 3.0 Omni のカスタム複数カット機能 + 参照画像 + カット毎指示で 1ワークフローに集約できる
  • 投稿者本人「仕事で背景問題に直面していて抜群のタイミングだった」と評価。実務適用直結の知見
